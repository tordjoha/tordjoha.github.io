The meaning of secure software

  • etymology: without risk, riskfree
    • not useful as definition, since its negative
  • DEF: /software security is the ability to function according to intentions in an adverserial environment/
    • Adverserial environment: our program is only a small part of the bigger sea, the adverserial environment.

Examples of assumptions

Assumptions pointing towards problems:

  • user input cannot be trusted to have property X
  • IP addresses can be spoofed
  • Computer resources are finite
  • Programmers write bugs

Assumptions pointing towards solutions:

  • if the program checks the input, we know it has property X
  • an attacker cannot guess a random 128 bit number
  • the semantics of the program
  • the type checker is correct
  • internet routing is quite robust

Sequrity requirements

Examples of requirements:

  • availability of the service
  • capacity
  • integrity of data
  • authenticity of data
  • recoverability

Sequrity mechanisms

Examples of mechanisms:

  • choice of programming languages
  • rate limiting
  • sanity checks on user inputs
  • access control lists
  • optimisaton of algorithms
  • encryption

Software security

  • Put together we get: assumptions + security mechanism = sequrity requirements

Threat model

  • the threat model explicates our assumptions about the system:
    • what threats applies to each component?
    • what are the trust relationships betweeen components?
    • which threats apply to each relationship?

Example:

Most transport encryption has requirements: confidentiability, integrity, authenticity. And assumptions:

  • attacker can read the messages when passing between sender and recipient
  • attacker cannot access messages after decryption

Either:

  • attacker cannot alter messages (passive attacker)
  • attacker can alter messages (active attacker)

Functional decomposition

For a specific program or system, a functional decomposition will help identify threats and clarify the relationship between different pieces of the system
