Sixth Lecture - Access control

What is access control?

Access control decides which users can

  • read/write to objects
  • perform operations
  • grant / revoke access

We are studying logical aspects of access control.

Logical aspects

  • An access control system consists of: subjects, objects and actions
  • and a model for how to relate these things (permissions)

Two axes of access control

  • classified according to:
    • mandatory / discretionary
    • permission model:
      • access control list
      • role based
      • capability based

Models are equivalent staticially, but have different dynamic behaviors.

Discretionary and mandatory

  • In a mandatory access control system: the access control policies are fixed by a central authority
  • In a discretionary access control system a subject who has access to an object can specify permissions for it or transfer access to another actor

Examples of mandatory access control

Modern operating systems have mandatory access control on resources such as CPU, memory and storage.

Examples of discretionary access control

  • file systems, e-mail, wifi passwords etc

Access control models

Access control lists

  • permissions are assigned to objects
  • each object has a list of permissions assigned to different users

Role based access control

  • a set of roles abstrac the permissions from users
  • we have sets P(permissions), R(Roles) and U(Users)
  • actions arae always performed by a role. To change role user must reauthenticate.

Capability based access control

  • users have capabilities
  • a capability consists of:
    • a reference to an object
    • a set of permissions for that object
  • a capability is used whenever a resouce is accessed.
